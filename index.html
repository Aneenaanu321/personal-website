Machine Learning Fairness 6 pages

Fairness-Aware Student Dropout Prediction: A Machine Learning Pipeline with Explainability and Bias Mitigation

Aneena Antony · Guided by Prof. Dr. Jarin T · Jyothi Engineering College, Thrissur, Kerala, India

Dropout risk tools often report a single accuracy score without showing who gets flagged. This work analyzes the Portuguese UCI student dataset (N=4,424) with four classifiers, Fairlearn group fairness tables, SHAP explainability, and gender bias mitigators. XGBoost achieved the best macro-F1 (0.711) and accuracy (77.6%). Scholarship aid status showed larger fairness gaps than gender. The paper argues shared dropout benchmarks should report who receives alerts, not only overall correctness.

XGBoost SHAP Fairlearn Educational Data Mining

Security Case Study 6 pages

From Feature-Complete to Production-Ready: A Case Study of Security and Quality Remediation in a Full-Stack Web Application

Aneena Antony · Guided by Prof. Jarin T · Department of Computer Science and Engineering, Jyothi Engineering College

Passing tests is not the same as being production-ready. This case study follows an Employee Management System built with React, Node.js/Express, and five-role RBAC. After a green CI build, a manual review uncovered five latent issues spanning JWT storage, missing auth/RBAC tests, non-transactional file storage, incomplete access rules, and refresh-token renewal. Each issue is mapped to OWASP Top 10 categories and fixed with version-control evidence, culminating in a twelve-item pre-deployment checklist.

OWASP RBAC React Node.js Code Review
